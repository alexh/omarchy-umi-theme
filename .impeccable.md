# Design Context

Codified context for `/impeccable` and related design skills working on this repo in future sessions. If you're a fresh agent landing here, read this before touching design decisions — it'll save you from re-deriving the rationale.

## Users

Omarchy / Hyprland / Arch users who install boutique themes — designers, creators, developers who curate their desktop deliberately. They pick this over defaults because they recognize the UMI (Utility Materials Inc.) aesthetic and want cross-surface consistency with other UMI themes (Blender, VS Code, Obsidian).

Context of use: long sessions on a personal machine. Job to be done: have a desktop that reads like a coherent industrial product, not a swatch of colors applied to defaults.

## Brand Personality

**Industrial. Utilitarian. Deadpan.**

UMI's voice is Carhartt product tags, 3M safety manuals, Home Depot orange aprons — industrial goods that describe themselves in flat, factual language. Emotional goal: quiet competence, never enthusiasm.

**Anti-tones**: edgy, jokey, shock-value, cyberpunk-ironic, promotional language ("beautiful!", "gorgeous!"), cutesy, playful.

## Aesthetic Direction

- **Theme**: dark only. No light variant in this repo. Omarchy ships light/dark as separate repos by convention.
- **Core palette**:
  - Background: `#1a1a1a` (industrial flat)
  - Foreground: `#fbf1c7` (cream)
  - Accent: `#FF6700` (UMI safety orange, canonical across all UMI products)
- **Secondary palette** (terminal syntax only, gruvbox-derived): `#fb4934`, `#b8bb26`, `#fabd2f`, `#83a598`, `#d3869b`, `#8ec07c`.
- **References**: Blender's Utility_Materials theme (warm gruvbox on `#3c3836`), VS Code's Utility Materials Protocol (flat industrial + `#B19CD9` purple secondary), UMI vault canon at `~/dev/vaults/UMI/08-Design-Library/`.
- **Anti-references**: synthwave neon, glassmorphism, purple-to-blue gradients, glossy tech-company minimalism, cyberpunk, generic "dark + blue accent" IDE themes.

## Design Principles

1. **Orange is activation, never decoration.** If orange appears in 8 UI elements at idle, it's wallpaper — exactly what the brand forbids. Reserve it for selection, focus, alarm. btop's two gradient classes (activation vs ambient) are the canonical application of this rule.
2. **Dark is structure; cream is the hero.** Chrome uses dark + warm grey. Cream (`#fbf1c7`) is rationed for high-contrast moments: foreground text, active selections, filled ambient meters.
3. **Terminal secondaries are quarantined.** Red / green / yellow / teal / pink / aqua live in the 16-color terminal palette for syntax highlighting only. They must not leak into Waybar, Hyprland borders, Mako, Walker, or any other desktop chrome.
4. **Fidelity over variety.** A 3-hex-point mismatch between Neovim (`#1d2021`) and Alacritty (`#1a1a1a`) backgrounds is a real bug. Prefer pixel-perfect consistency across surfaces to ecosystems of similar-but-not-identical colors.
5. **Every file is a design artifact.** README, btop header comments, LICENSE placement are all opportunities to assert brand through restraint. Plumbing files should read like spec sheets, not generators' output.
6. **Recommended pairing**: Monaspace Krypton as the mono font. Omarchy doesn't override fonts (that's Alacritty's config), but this is the intended pairing — document it in README, don't hardcode it.
